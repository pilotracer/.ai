# Quick-reference cheat sheets (`.quick/`)

**Location:** `.ai/.quick/` (or `.quick/` at repo root when Agent OS is the git root)  
**Purpose:** Copy-paste command snippets for the most common Agent OS workflows — no need to open full skill docs for routine tasks.

Each file is a focused, one-page reference for a specific workflow pattern:

| File | When to open |
|------|--------------|
| [`add-feature-workflow.md`](add-feature-workflow.md) | Taking a feature idea from intake through implementation |
| [`explore-codebase.md`](explore-codebase.md) | Understanding how specific code behavior works — tracing logic, calculations, business rules |
| [`brownfield-onboarding.md`](brownfield-onboarding.md) | First time working with an existing (non-greenfield) repo |
| [`db-migration.md`](db-migration.md) | Creating, running, or verifying database migrations |
| [`plan-implementation-create.md`](plan-implementation-create.md) | Planning an implementation iteration (NEXT.md block) |
| [`repair-and-fix.md`](repair-and-fix.md) | Fixing verifier findings, plan drift, or test failures |
| [`session-lifecycle.md`](session-lifecycle.md) | Starting, checking, or closing a session |
| [`verify-all-levels.md`](verify-all-levels.md) | Verification at every stage — pre-commit, post-commit, milestone, plans |
| [`verify-committed-code.md`](verify-committed-code.md) | Auditing committed code and adding tests |

**Not a replacement for skill docs.** For full protocols, gates, and edge cases, open the corresponding skill under `skills/<id>/skill.md`.
