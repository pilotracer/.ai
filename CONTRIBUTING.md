# Contributing to Agent OS

Thanks for considering a contribution. Agent OS is a **portable process framework** — almost everything is Markdown.

## Scope of contributions

| Welcome | Out of scope |
|---------|--------------|
| New or improved **skills** under `skills/<id>/skill.md` | Project-specific (vendor / company) artifacts |
| Engineering **standards** templates under `standards/` | Tool-specific lock-in (Cursor-only flags, vendor APIs) |
| **Workflow guides** under `docs/guides/workflows/` | Generated content (AI attribution, "Created by …") |
| **Concept** prompts under `concepts/<id>/` | Backwards-incompatible skill renames without migration notes |
| Bug fixes / typo fixes anywhere | Adding `AGENTS.md` (use `.cursorrules` per project decision) |

## Quick checklist before opening a PR

1. **Read** [`.cursorrules`](.cursorrules) § Core Principles (truth-first, evidence-first, completion gate).
2. **Follow naming protocol** for new skills: `{domain}-{role}` kebab-case ([`skills/README.md`](skills/README.md) § Naming protocol).
3. **Update cross-links** when renaming: `.cursorrules`, `skills/README.md`, `skills/SKILL_DEPENDENCIES.md`, `process-router/reference.md`, README "Skills at a glance".
4. **Test bootstrap** from a clean directory: `bash templates/bootstrap.sh` should succeed.
5. **Verify links:** no references to deleted paths (e.g. removed `docs/integration/*` vendor artifacts).
6. **No AI attribution** in any added text ("Generated by", signatures, dates).
7. **Commit message format** per `.cursorrules` § Commit Message Format (plain text, no surrounding quotes).

## Adding a new skill

```text
skills/<skill-id>/
├── skill.md       # required — YAML frontmatter + protocol
└── reference.md   # optional — extended examples
```

Register the skill in:
- `skills/README.md` § Registered skills
- `skills/SKILL_DEPENDENCIES.md` (prerequisite gate)
- `README.md` § Skills at a glance
- `process-router/reference.md` (routing row, if user-facing)

## Reporting issues

Open a GitHub issue with:
- Which **skill** or **standard** is affected
- What the agent or human **did**
- What the agent or human **expected**
- Repro steps (chat invocation, file paths, exit codes if applicable)
